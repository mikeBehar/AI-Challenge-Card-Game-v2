## Purpose

**Not for students** - This is instructor-facing material for course development.

## Development Workflow

### Creating a New Case Study
1. Review requirements in `/requirements/requirements.md`
2. Copy template from `/templates/` (when available)
3. Draft case study following established structure
4. Create assignments aligned with rubrics
5. Develop ideal answers for grading reference
6. Test with students and iterate
7. Document lessons learned

### Continuous Improvement
- After each case delivery, update LESSONS-LEARNED.md
- Move completed items from TODO.md to DONE.md
- Add new insights to CONVERSATION-CONTEXT.md
- Update requirements based on what works/doesn't work

## Course Philosophy

This course emphasizes:
- **AI collaboration skills** over domain expertise
- **Business judgment** and practical thinking
- **Ethical awareness** as business consideration
- **Deliberation** and engaging with diverse perspectives
- **Iterative learning** through case-based instruction

See requirements.md for full pedagogical framework.
