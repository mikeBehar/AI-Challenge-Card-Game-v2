## Framework Reference

### The 6-Part AI Risk Assessment Framework

**1. Data Quality Risks**
- Is training data representative?
- Does historical data contain biases?
- Are proxy variables valid for all groups?

**2. Transparency & Explainability Risks**
- Can the system explain decisions?
- Can we audit for fairness?
- Can users challenge decisions?

**3. Testing & Validation Risks**
- Pre-launch fairness audits?
- Edge case testing?
- Continuous monitoring?

**4. Governance & Oversight Risks**
- Clear accountability?
- Human review for high-stakes decisions?
- Mechanisms to report harms?

**5. Scope & Safeguards Risks**
- Clear boundaries on authority?
- Safety filters in place?
- Escalation pathways?

**6. Context & Equity Risks**
- Diverse perspectives in design?
- Understanding of social context?
- Who benefits vs. who bears risks?

When creating materials:
- Use high contrast colors
- Clear, readable fonts (min 11pt)
- Logical heading structure
- Alt text for images
- Screen reader compatibility
